# Privacy Policy — GSA Gateway

*Effective: May 2026 | Maintained by: GSA VP Academic Affairs*

## What We Collect

GSA Gateway collects only the minimum data necessary to operate:

| Data | Purpose | Retention |
|---|---|---|
| SHA-256 hash of Discord user ID | Rate limiting, deduplication | Indefinitely (no personal info) |
| Question text | Improve knowledge base | Indefinitely |
| Feedback messages | Officer review | Indefinitely |
| Initiative title, description, category | Officer review and tracking | Indefinitely |
| Contact info (optional, opt-in only) | Follow-up on initiatives | Until fulfilled or deleted on request |
| Discord guild ID | Multi-server support | Indefinitely |

## What We Do NOT Collect

- Raw Discord user IDs — all IDs are immediately hashed with SHA-256 before storage
- Discord usernames or display names
- Message history outside of explicit bot commands
- Any data from channels where the bot has not been used

## How Data Is Stored

- All data is stored in a local SQLite database on the maintainer's machine
- The database file is excluded from version control (`.gitignore`)
- Exports generated by `/admin_export` are sent as ephemeral attachments visible only to the requesting officer

## Data Sharing

- GSA officer data is reviewed only by GSA officers with the `GSA Officer` Discord role
- Data is not shared with third parties, sold, or used for advertising
- If the bot is enabled with Ollama, question context is sent to a locally running language model — no data leaves the server

## Your Rights

- You may request deletion of any data you submitted by contacting the GSA VP Academic Affairs
- Since user IDs are hashed, we cannot definitively link historical data to a specific user without additional information you provide
- Anonymous submissions (feedback, questions) cannot be linked back to you by design
