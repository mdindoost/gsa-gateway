# Phase 0 — v1→v2 Boundary Cut Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make v2 the sole originator of any outbound message — disable every v1 autonomous sender and trim the built-in Discord command surface to `/help` + `/contact` — without breaking the transport, the `#ask-gsa` RAG responder, or the v2 scheduler.

**Architecture:** The functional cut is purely *unregistration*: remove the offending entries from `EXTENSIONS` in `bot/main.py` and stop instantiating the now-orphaned MathCafe service. No source files are deleted in the functional cut, so all unit tests stay green and the change is instantly reversible. A separate, gated cleanup task (Task 6) deletes the dead files only after the live cut is verified. A mandatory, un-skippable backup precedes any edit.

**Tech Stack:** Python 3.11, discord.py 2.x, pytest. Live process: `.venv/bin/python -m bot.main` (currently PID 549083).
