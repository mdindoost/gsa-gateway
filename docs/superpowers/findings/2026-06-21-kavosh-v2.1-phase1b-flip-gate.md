## Build deviations from the literal plan (documented)

1. **`_answer_decision` returns a `MessageResponse`, not a bare string** (the plan literally wrapped
   `MessageResponse(text=await _answer_decision(...))`). Returning the full response preserves RAG/LIVE
   source-notes, feedback buttons, and the live flag — the literal text-wrap would silently drop them
   (a regression against the plan's zero-regression goal). `handle` returns it directly.
2. **COMMAND family is handled by the legacy intent flow, not duplicated in `_answer_decision`.** In ACT
   mode a COMMAND decision falls through to the existing command branches (same `IntentDetector` →
   identical handling), avoiding a risky copy of the large greeting/identity blocks. Behavior-identical.
3. **Example tests needed `platform=` on `MessageRequest`** (a required field) and `@pytest.mark.asyncio`
   instead of `asyncio.run`/`get_event_loop` (the latter closed the shared loop and broke worldcup
   tests). Functionally equivalent; matches the repo's async-test convention.
4. **Startup batch-embed** folded into D3 (Embedder `_embed_batch` + `route_exemplars._encode_prefixed`)
   to satisfy the S-2 "no silent multi-second startup" requirement.
