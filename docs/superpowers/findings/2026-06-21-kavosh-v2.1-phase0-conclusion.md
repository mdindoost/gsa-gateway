# Kavosh v2.1 — Phase 0 Conclusion (router bake-off)

**Date:** 2026-06-21 · **Branch:** `feat/kavosh-v2.1-phase0-router-bakeoff` (not merged) ·
**Status:** Phase 0 COMPLETE — architecture decided with data. Phase 1 build is under the HARD GATE.

## The question Phase 0 had to answer
Should Kavosh v2.1's router be **detector-first** (today's regex), **coarse-classify-then-deterministic**,
or a **full 18-skill classifier** — decided by data, not priors, with a blocking anti-fabrication gate?

## Answer
**Masked coarse-family classifier → deterministic skill selection.** A learned embedding classifier
(nomic exemplar-NN, entity **slot-masked** before encoding) picks the FAMILY; the existing deterministic
resolver picks the KG skill and fills slots. The **full skill-classifier is rejected** — it leaks
(wrong-confident-exact dispatch) even with 5× the training data. This matches the spec's original prior,
now backed by 97 blind-labeled, IAA-checked real gold rows across five growth points (28→56→76→101→97).

## Final bake-off (entity-disjoint primary split, 514 train / 97 gold)
| Arm | family | skill | anti-fab (wce) | gate |
|---|---|---|---|---|
| detector_first (today) | 0.577 | 0.91 | 2 | baseline |
| coarse_then_deterministic | 0.649 | 0.90 | 2 | ✅ pass |
| **masked_coarse** | 0.649 | **0.95** | **1** | ✅ pass |
| full_classifier | 0.763 | 0.78 | 7 | ❌ reject |
| masked_full | 0.794 | 0.83 | 6 | ❌ reject |

## Key findings (including the useful negatives)
1. **Skill selection must stay deterministic.** Both full-classifier arms leak and are gate-rejected
   across every run; coarse arms (skill from the resolver) stay anti-fab-clean. Masking lifts the full
   classifier's skill accuracy but never enough to clear the gate.
2. **Masking is the real lever, not augmentation.** Slot-masking the entity (`<ORG>`/`<PERSON>`) before
   encoding removes the dominating-token bias; it improves skill accuracy and cuts leaks. Template×entity
   augmentation was explicitly rejected by the panel of advisors (adds more entity-dominated vectors).
3. **More training data does NOT crack the family ceiling.** 113→514 exemplars moved coarse family
   accuracy only 0.634→0.649 (masked_coarse flat). The ~0.65 ceiling is **not** a data-volume problem.
4. **The bottleneck is the KG-vs-RAG boundary.** ~18 structured-false-negatives = KG/CLARIFY questions
   dropped to RAG (the majority class). Adding RAG-heavy train data can reinforce that bias.
5. **Abstention is inactive only because it's mis-calibrated.** Calibrated on TRAIN (precision trivially
   ~100% → margin 0), so it never fires — yet the TEST leaks 6. Calibrating on a held-out VAL split is the
   fix and a Phase-1 lever, not a Phase-0 defect.

## Phase-1 levers (explicitly NOT "more data")
1. Improve the **KG-vs-RAG family signal** / address class imbalance (the 18 structured-FN).
2. **Calibrate abstention on a VAL split** so low-confidence family calls route to CLARIFY, not a wrong
   RAG drop. (Mechanism already built — `abstain.py` — just needs val calibration.)
3. Keep **skill selection deterministic + slot-masking** (validated repeatedly).
4. (Phase 2) A **single-turn → context-rewrite** step to resolve the `hardneg` follow-ups (the 23+ rows
   we set aside are the ready-made test set for it).

## Method that produced the gold set (and the trust guarantees)
- **Blind gold:** Mohammad blind-labels harvested real questions; an **independent annotator** labels the
  same set; **Cohen κ** + adjudication of disagreements; rules sharpened each batch. 97 gold + 23 hardneg.
- **Expert panel (train labels):** 3 Claude experts (router-eng / knowledge-arch / intent-UX) voted blind
  on all 571 distinct questions → **Fleiss κ=0.93**; 41 disagreements → Round-2 debate → **0 deadlocks,
  6 ratified rules** (R13–R15 etc., in `eval/router/RUBRIC.md`). 401 routable rows → TRAIN.
- **Locked methodology:** **human votes are GOLD, the panel owns TRAIN.** The 3 experts are all Claude →
  κ=0.93 is self-consistency, not ground truth; the human vote is the only non-circular anchor. The panel
  may never grade itself. True-independence upgrade = a non-Claude 2nd gold annotator (deferred).
