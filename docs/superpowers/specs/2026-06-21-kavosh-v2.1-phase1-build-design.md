## 0. What Phase 0 already decided (do not re-litigate)
- **Architecture (data-backed):** masked coarse-family classifier → deterministic skill selection.
  Full 18-skill classifier REJECTED (leaks wrong-confident-exact even at 5× data).
- **Methodology (locked):** human votes = GOLD (97 blind rows + 23 hardneg); 3-Claude panel = TRAIN
  (514 rows, Fleiss κ=0.93); the panel may never grade itself.
- **The ceiling is NOT a data problem.** 113→514 exemplars moved family accuracy 0.634→0.649. The
  bottleneck is the KG-vs-RAG boundary (~18 structured-false-negatives) and inert abstention.

## 1. Goal of Phase 1
Ship the production router from the FINAL spec — but de-risk the central bet FIRST. The expensive part
(production integration: command layer, fast-path, `query_vec` seam, resolver, shadow infra) is only
worth building if the learned family classifier is meaningfully better than today's regex router. Phase 0
proved it is *already* a Pareto win; Phase 1a tries the two named levers to widen that margin before we
pay for the plumbing in Phase 1b.

## 2. Overall shape — two sub-phases with an explicit gate
- **Phase 1a — offline lever sprint.** ZERO production code; all work under `v2/eval/router/` +
  `scripts/`. Exhaust the two levers from the conclusion until the masked-coarse arm clears the §3 gate
  OR provably plateaus. Deliverable: updated `eval/router/bakeoff_report.md` + a go/no-go writeup.
- **Phase 1b — production integration.** The FINAL spec §10 Phase-1 build, behind `ROUTER_V21`,
  shadow-mode first. Begins ONLY after the 1a gate (§3).

## 3. The 1a → 1b gate (the new decision artifact)
The gate has one HARD CONSTRAINT, one TARGET, and a FLOOR that means it never blocks the integration:
- **Hard constraint (per lever):** every lever must stay gate-clean — no NEW `wrong_confident_exact` or
  `false_honest_partial` on the held-out **gold** vs the detector-first baseline. A lever that introduces
  an anti-fab regression is rejected outright (dropped from the shipped config — it does not pause 1b).
- **Target we push the levers toward:** widen the masked-coarse arm's win over detector-first on BOTH
  family accuracy AND `structured_false_negative`.
- **Floor guarantee — why the gate never blocks 1b.** The masked-coarse arm ALREADY beats detector-first
  on the gold across the board (family **0.649 vs 0.577**, skill **0.95 vs 0.91**, anti-fab wce **1 vs
  2**). So even if both levers fully plateau, Phase 1b still proceeds — the integration is a net Pareto
  win on its own. The levers decide *how much* better the shipped router is, not *whether* we integrate.

**Net:** the only thing 1a can "fail" is an individual lever (→ ship masked-coarse without it). Phase 1b
proceeds in every case; the go/no-go writeup just records which levers made the cut.

## 4. Phase 1a — the two levers (concrete, offline, measured)
Ordering: Lever #2 first (cheaper, higher-confidence, mechanism already built), then Lever #1.

### Lever #2 — VAL-calibrated abstention
**Problem (from the conclusion):** abstention is inert because `calibrate_thresholds` was run on TRAIN,
where exemplar-NN precision is trivially ~100% → margin threshold 0 → never fires; yet the gold TEST
still leaks ~6.
**Work:**
1. Add a 3-way split helper (`train / val / test`) to `split.py`. `val` is carved from the TRAIN pool
   (the 514 panel rows); **gold stays the untouched test set**. The entity-disjoint guarantee
   (`split_entity_disjoint` / union-find components) is preserved across all three folds — no entity or
   paraphrase group straddles a boundary.
2. Calibrate `abstain.py` margins on **val**, evaluate on **gold**.
3. **Success criterion:** abstention reports ACTIVE (not INACTIVE/DEGENERATE — the loud flag already
   exists in `calibrate_thresholds`), fires on low-margin gold rows, and reduces the gold leak count at
   an acceptable clarify-precision cost (clarify precision measured on gold; a low-margin KG question
   routed to CLARIFY instead of a wrong RAG drop is the intended behavior).

### Lever #1 — KG-vs-RAG family signal
**Problem:** ~18 structured-false-negatives — KG/CLARIFY questions classified as RAG (the majority
class). RAG/general swamps the exemplar vote.
**Bounded experiments (each measured by `structured_false_negative` on gold, with the §3 anti-fab gate
as a hard constraint; an experiment that regresses anti-fab is dropped):**
1. **Class-balance the exemplar vote** — cap / down-weight the over-represented RAG/general exemplars so
   KG recall isn't drowned.
2. **KG-recall-biased decision** — a per-family margin/threshold, or a two-stage "KG-vs-rest" gate, that
   prefers running the deterministic resolver when KG is a close runner-up. This is the OFFLINE analogue
   of the spec's §5 step-4 inverse-FN guard; if it works here it pre-validates that production guard.
3. **Commit the masked-coarse arm into `arms.py`** as a first-class, reproducible arm (today masking is
   only an encoder swap via `mask.py`; the committed arms are detector_first / coarse_then_deterministic /
   full_classifier). The winning configuration must be a named arm the bake-off runs every time.

**Kept only if:** the experiment cuts `structured_false_negative` without hurting family/skill accuracy
and without any anti-fab regression. Negatives are recorded in the report (useful negatives, per the
conclusion's style), not silently dropped.

## 5. Phase 1b — production integration (from FINAL spec §10, two refinements folded in)
Unchanged in SHAPE from the approved architecture spec §5; the bake-off only fixes two parameters:
- **The classifier IS the masked-coarse arm** (entity slot-masked before nomic encode — `mask.py`), not
  a raw-text exemplar-NN.
- **Abstention uses the val-calibrated thresholds** from Lever #2 (not a hand-picked floor).

Scope (each becomes a plan task; this is the §10 Phase-1 list, restated so the plan can check it off):
1. One unified Router module: COMMAND layer (deterministic whole-message) → first-class FAST-PATH →
   masked coarse CLASSIFIER → deterministic RESOLVER/VALIDATOR → CLARIFY / 3 RAG outcomes / live.
2. Resolver re-runs `route()`'s NEGATIVE GUARDS as dispatch preconditions (`_OFFICER_PROCESS` /
   `_LEADERSHIP_PROCESS` / child-departments / bare-name / ≤4-tok surname), the TERMINAL-SKILL guard
   (link/metric require the regex precondition), and the INVERSE-FN guard (§5 step 4).
3. The `query_vec` seam through `V2RetrieverShim.retrieve` → `V2Retriever.retrieve` + the ONE-encode-
   per-message invariant (unit test asserts `embed_query` ≤ 1× per `handle()`) + the encode-prefix/
   version guard (raw router prefix; startup fails loudly on `EMBEDDING_MODEL`/`OLLAMA_URL` drift).
4. First-class fast-path sized from the Phase-0 head distribution + a p95 latency budget/go-no-go gate.
5. `ROUTER_V21` flag (default OFF) + shadow-mode logging (new-vs-current on real traffic + the labeled
   set) + the conjunctive flip gate (spec §12). Flag stays as kill-switch.
6. Anti-fab invariants preserved VERBATIM: `is_deterministic` + `deterministic_suffix` skip-compose path
   (`message_handler` ~L317-336); empty result → RAG (never a fabricated honest-partial).
7. Phase 2 (LLM slot-recovery, sub-flag) stays OUT of Phase 1 — explicitly deferred, zero-regression
   when off.

## 7. Testing (TDD)
- **Phase 1a:** harness unit tests for the 3-way split (entity-disjoint preserved), val-calibration
  (ACTIVE/INACTIVE flag), the new masked-coarse arm, and each lever experiment. Bake-off re-run is the
  integration test.
- **Phase 1b:** unit tests per component (command precedence, fast-path, classifier accept/margin,
  resolver slot-validation + candidate-IDs-only + empty→RAG, negative-guard preconditions, terminal-skill
  guard, inverse-FN guard, clarify templates, one-encode invariant). The regression set runs THROUGH
  `handler.handle` (NOT `scripts/ask.sh` — it bypasses the gate; assert this). The known-failing live set
  ("top 10 by citations in mechanical engineering", role/area paraphrases) + the negative-guard cases
  become committed regressions. Every change grows `eval/questions.txt`.

## 8. Non-goals (Phase 1)
No LLM on the hot path (Phase 2). No text-to-SQL. No new heavy dependency / second embedding model / fine-
tuned head. No fine-grained within-RAG sub-routing (deferred, flagged). No 18-skill classifier (rejected
by Phase 0). No merging the Phase-0 branch as a precondition (the harness work continues on it).
